# React Doctor Audit & Fix Plan

## Project Overview
- **Framework**: TanStack Start (React 19) with TypeScript
- **Linting**: Biome + Ultracite
- **Testing**: Vitest
- **Package Manager**: Bun

## Goals
1. Apply React Doctor skill to catch React-specific issues
2. Run `bun run fix` (ultracite) to auto-fix issues
3. Run audit: lint, typecheck, tests
4. Refactor any remaining issues
5. Commit changes

## Execution Steps

### Phase 1: React Doctor Analysis
Review components for:
- Missing dependencies in useEffect/useCallback
- Incorrect dependency arrays
- Derived state issues
- Handler stability problems
- Event listener cleanup
- Stale closure issues
- Missing React import (React 19 JSX doesn't require it, but some files have it)
- Unused imports/variables

Files to focus on:
1. `src/routes/__root.tsx` - Has React imports that may be unnecessary
2. `src/components/evilcharts/charts/bar-chart.tsx` - Complex hooks with eslint-disable comments
3. `src/lib/api.ts` - Large file with potential unused code
4. `src/components/layout/app-shell.tsx` - Sidebar state handlers
5. `src/routes/dashboard.tsx` - State management

### Phase 2: Auto-Fix
- Run `bun run fix` (ultracite) to fix formatting and auto-fixable issues
- This will handle:
  - Import organization
  - Code formatting
  - Simple lint fixes

### Phase 3: Lint & Type Check
- `bun run lint` - Biome check
- `bun run typecheck` - TypeScript validation
- `bun run test` - Vitest tests

### Phase 4: Manual Refactoring
Fix any remaining issues from lint/type check:
- Unused variables/parameters
- Missing type annotations
- Hook dependency issues
- Remove unnecessary React imports (React 19 JSX transform)

### Phase 5: Verification
- Ensure all tests pass
- Verify build succeeds: `bun run build`
- Confirm no lint errors

### Phase 6: Commit
- Create commit with descriptive message
- Include: "fix: react doctor audit - fix lint/type issues"
